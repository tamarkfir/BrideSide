# BrideSide — Claude Code Brief

## מה זה BrideSide?

אפליקציית ווב שמנחה זוגות לסביים לבנות טקס חתונה אישי — דרך AI מנחה שמקשיב, מחבר ובונה מהחומרים שהזוג מביאה. הפגישה נמשכת שעה עד שעתיים. הכל רץ על מחשב אחד (בשלב הזה — בלי סנכרון טלפונים).

---

## המטרה הראשונה — MVP

בנה גרסה ראשונה שעובדת מקצה לקצה:
- AI מנחה שמוביל את הזוג דרך 6 שלבים
- בכל שלב — חלקים קריטיים בלבד (לא הכל)
- בסוף — פלט: "ספרון הטקס" שמרכז את מה שנבנה
- עיצוב נקי, אורגני ויפה — ראה הנחיות עיצוב למטה

---

## סטאק טכנולוגי

- **Framework:** Next.js (App Router)
- **Styling:** Tailwind CSS
- **AI:** Anthropic Claude API — `claude-sonnet-4-6`
- **Language:** TypeScript
- **State:** React useState / useReducer (לא Redux בשלב זה)
- **אין צורך ב-DB בשלב זה** — state בזיכרון בלבד

---

## ארכיטקטורת המסכים

בשלב זה — מחשב אחד, שני אנשים יושבים ליד אותו מסך.
הזוג מחליפות בין עצמן מי כותבת (לפי הנחיית ה-AI).
**סנכרון טלפונים יתווסף בגרסה הבאה** — בנה את הקוד כך שיהיה קל להוסיף.

## מבנה השלבים — מה לממש ב-MVP

### שלב 1 — "מי אנחנו" (פתיחה)
**קריטי לממש:**
- ה-AI פותח בברכה חמה ומסביר את התהליך
- כלה א׳ כותבת עד 5 מילים — איך היא מרגישה לגבי החתונה
- כלה ב׳ כותבת עד 5 מילים — איך היא מרגישה
- שתי התשובות מוצגות זו לצד זו — רגע גילוי
- ה-AI מגיב בחמימות לשתי התשובות ביחד
- כל אחת מספרת על רגע ההיכרות ועל "משהו שהשנייה לא יודעת"
- העלאת 3 תמונות לכל אחת + טקסט מסביר (תמונות נשמרות ל-state, לא חייב להציג)
- כל אחת מכניסה שיר או תחושה של שיר

**פלט השלב:** "כרטיס היכרות" — שמות, 5 המילים, משפט שה-AI כתב על הזוג

---

### שלב 2 — "מאיפה אנחנו באות" (שורשים)
**קריטי לממש:**
- כל אחת כותבת על הרקע שלה (תרבות, דת, משפחה) וזיכרון מחתונה
- כל אחת מכניסה חופשי: אלמנטים שרוצה, סגנון, השראות (לאו דווקא יהודיות)
- ה-AI בונה ומציג "קולאז' כוונות" — רשימת מילות מפתח וביטויים שעלו, מסודרים בצורה ויזואלית (לא חייב להיות תמונות — טקסט מעוצב מספיק ב-MVP)
- ה-AI שואל עד 2 שאלות על פערים שזיהה

**פלט השלב:** קולאז' הכוונות — נשמר ל-state

---

### שלב 3 — "מה בינינו" (DNA של הקשר)
**קריטי לממש:**
- ה-AI חוזר לחומרים משלב 1+2 ומציג 4-5 "ערכי ליבה" בשפת הזוג
- הזוג מאשרת / מתקנת / מוסיפה
- ה-AI שואל שאלת NVC אחת על פער שזיהה

**פלט השלב:** ערכי הליבה — 4-5 משפטים

---

### שלב 4 — "השירים והמילים שלנו"
**קריטי לממש:**
- ה-AI לוקח את השירים משלב 1 ומציע ציטוט רלוונטי לטקס + הסבר למה
- ה-AI מציע 2 שירים נוספים מתאימים (על בסיס הרקע והערכים) + ציטוט + סיבה אישית
- הזוג בוחרת מה נכנס לטקס

**פלט השלב:** ספריית מילים — ציטוטים שנבחרו

---

### שלב 5 — "הטקס שלנו" (הספרון)
**קריטי לממש:**
- ה-AI בונה טיוטת טקס מכל מה שנאסף
- מוצג כ"ספרון" — עמודים שאפשר לדפדף
- על כל עמוד אפשר להוסיף הערה
- עמוד אחרון: 2-3 תרגילי כתיבה מותאמים אישית שה-AI מייצר

---

### שלב 6 — "כוונה" (סגירה)
**קריטי לממש:**
- כל אחת כותבת משפט לשנייה — "משהו שגילית היום"
- המסך מציג את שתי ההודעות
- ה-AI מסיים בברכה אישית
- כפתור "הורד ספרון" — מייצר PDF או עמוד מודפס

---

## ה-AI — System Prompt

השתמש ב-system prompt הזה לכל קריאות ה-API:

```
אתה מנחה של BrideSide — חוויה דיגיטלית שעוזרת לזוגות לסביים לבנות טקס חתונה אישי.

אישיותך:
- חמה אבל לא סוכרית. לא אומרת "נפלא!" או "מדהים!" — אומרת "אני שומעת את זה."
- מקשיבה לרגש לפני שמגיבה לתוכן
- משתמשת בשמות של הכלות — לא "הזוג" כשפונה אישית
- מצטטת מה שנכתב: "כשכתבת ___ הרגשתי ש..."
- לא מכריעה — מזמינה. כשיש מתח, שואלת על צרכים לא על עמדות
- לא מציעה יותר מ-3 אפשרויות בכל פעם
- עברית. משפטים קצרים. מרחב.

מה שאסור לך:
- לשפוט בחירות (לא "זה יפה", לא "בטח שכן")
- לתת דוגמאות שלא קשורות לזוג הספציפי הזה
- למהר לפתרון לפני שהקשבת

בכל תגובה — קרא קודם מה נאמר, זהה את הרגש, ואז ענה.
```

## עיצוב — הנחיות מפורטות

### עקרון מרכזי
**העיצוב חייב להיות קל לשינוי.** כל הצבעים, הפונטים, הרווחים — ב-`tailwind.config.ts` ו-`globals.css` בלבד. אף צבע hardcoded בקומפוננטות.

### פלטת צבעים — להגדיר ב-tailwind.config.ts
```
brand: {
  blush:    #F2E8E4   // רקע ראשי — ורוד חם מאוד בהיר
  rose:     #C47E8A   // accent ראשי — ורוד עמוק
  roseDark: #8B4F5C   // hover / כותרות
  sage:     #8A9E8B   // accent משני — ירוק-אפור
  sageDark: #5C7260
  sand:     #B5A090   // טקסט משני
  cream:    #FAF6F1   // כרטיסים / surfaces
  ink:      #2C2420   // טקסט ראשי
}
```

### טיפוגרפיה
- **פונט כותרות:** Cormorant Garamond (Google Fonts) — אלגנטי, אורגני
- **פונט גוף:** Inter — קריא, נקי
- **גדלים:** scale ברור — text-sm / text-base / text-lg / text-xl / text-3xl בלבד

### עקרונות עיצוב
- **הרבה whitespace** — הכל נושם
- **אנימציות עדינות** — fade in כשעמוד נטען, transition על כפתורים
- **כרטיסים** — `rounded-2xl`, `shadow-sm`, רקע `cream`
- **כפתורים** — rounded-full, padding נדיב, hover עדין
- **אין** borders חדים, אין צבעים זועקים, אין תיבות טקסט שטוחות
- **RTL** — כל הממשק בעברית מימין לשמאל, `dir="rtl"`

### Layout
```
- רוחב מקסימלי: max-w-2xl ממורכז
- padding צדדי: px-6 על מובייל, px-12 על דסקטופ
- בין אלמנטים: space-y-6 ברירת מחדל
- כרטיס AI: רקע שונה מעט מהרקע הראשי, border-r-4 בצבע rose
```

### קומפוננטות שכדאי לבנות כ-reusable
```
<AIMessage />       — הודעת AI עם סגנון ייחודי
<BrideInput />      — שדה קלט עם שם הכלה מעליו
<PhaseHeader />     — כותרת שלב עם מספר ושם
<StepCard />        — כרטיס שלב
<BookPage />        — עמוד בספרון
<OutputCard />      — כרטיס פלט בסוף שלב
```

## חשוב — מה לשמור גמיש

1. **כל הטקסטים של ה-AI** — ב-`/lib/prompts.ts` בלבד, לא inline בקומפוננטות
2. **כל הצבעים** — ב-`tailwind.config.ts` בלבד
3. **מבנה השלבים** — כל שלב קומפוננטה נפרדת, קל להחליף סדר
4. **System prompt** — קובץ נפרד, קל לעדכן את אישיות ה-AI
